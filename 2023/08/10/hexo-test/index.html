基于hexo搭建博客记录

第一次用 hexo，在此记录搭建过程中有用的 tips 和踩的坑
主题：freemind.bithack

常用命令

新建文章

$ hexo new [layout] <title>

生成静态文件

$ hexo generate
// 简写
$ hexo g

启动服务器

$ hexo server
$ hexo s

本地调试，支持热更新

$ hexo s -g --debug

部署网站

$ hexo deploy

hexo g 部署报错(部署到 github)

  1. 使用用户名+密码登录报错
    2021.8.13 开始对 git 操作不再支持通过账户+密码来验证，必须基于 token 验证
开个坑，后期有空补充
  1. hexo 配置问题
# repo 应为git仓库的ssh链接
deploy:
  type: git
  repo: git@github.com:github-username/github-username.github.io.git
  branch: master

点击导航显示无法访问的问题

错误情况

  • 使用命令创建 Tags，会在 source 下生成 tags/index.md，完成 layout 等配置后点击导航 Tags，此时跳转路径为 /tags，页面显示无法访问，查看该主题的 demo 发现跳转路径为 /tags/
  • 手动将地址栏的地址改为 /tags/，页面正常

修改尝试

  • 直接在 source 下创建 tags.md 文件，并添加下面内容
---
title: tags
date: 2023-08-10 11:16:46
type: 'tag'
layout: 'tags'
---
  • 此时点击导航正常跳转 (原因不明，后期找到原因再补充)

(部署后发现即使没有创建 archives 文件，跳转也不会报错，且路由后加上了’/‘，继续跟踪这个问题)
